# 후쿠오카 프로젝트

## 프로젝트 목표

후쿠오카 가족여행을 효율적으로 준비하고,
여행 중에도 쉽게 사용할 수 있는 노션 시스템을 구축한다.

여행 종료 후에는 이 노션을 기반으로 웹앱을 개발한다.

---

## 현재 단계

Release R1

노션 DB 구축 및 품질 향상

---

## 현재 우선순위

1. 쇼핑 DB 완성
2. 여행 일정 및 장소 점검
3. 여행 준비 완료
4. 후쿠오카 여행 (7월 29일 ~ 7월 31일)
5. 여행 후기 및 실제 구매 내역 반영
6. 웹앱 개발 시작

※ 웹앱 개발은 여행 종료 후 진행한다.

---

## 완료된 작업

- 쇼핑 DB 구축
- 중복 점검 완료
- Description 컬럼 작성 완료
- 대표이미지 컬럼 추가
- 검색어 컬럼 추가
- 카테고리 정리
- 구매처 정리
- 결정 상태 정리

---

## 진행 중

- 검색어 데이터 입력
- 대표이미지 등록

---

## 작업 원칙

- 새 DB 생성 금지
- 기존 DB만 수정
- DB 구조 변경은 승인 후 진행
- 기존 데이터 삭제 금지
- 중복 생성 금지
- 작업 전 계획 보고
- 작업 후 변경 내용 보고
- 기존 프로젝트 방향을 유지하며 작업한다.

---

## 프로젝트 결정 사항

- 노션을 기준 데이터베이스(SSOT)로 사용한다.
- 여행 전에는 노션 DB 품질 향상에 집중한다.
- 웹앱 개발은 여행 종료 후 시작한다.
- PROJECT.md를 프로젝트의 유일한 기준 문서로 사용한다.
- 새로운 관리 문서는 만들지 않는다.
